# ⚠️ OAuth 오류 해결: 접근 거부됨 (Error 403)

## 문제 원인

```
접근 차단됨: faq1은 Google 인증 절차를 완료하지 않았습니다.
오류 403: 접근 거부됨
```

OAuth 동의 화면이 **"테스트 모드"**로 설정되어 있고,
현재 로그인하려는 Google 계정이 **테스트 사용자 목록에 없습니다**.

---

## ✅ 해결 방법

### 1단계: OAuth 동의 화면 접속

https://console.cloud.google.com/apis/credentials/consent

### 2단계: 테스트 사용자 추가

#### 방법 A: 테스트 사용자 추가 (권장, 빠름)

1. **OAuth 동의 화면** 페이지에서 아래로 스크롤
2. **"테스트 사용자"** 섹션 찾기
3. **"+ ADD USERS"** (사용자 추가) 버튼 클릭
4. 로그인할 Google 계정 이메일 입력:
   ```
   kongziac@gmail.com
   ```
5. **"저장"** 또는 **"SAVE"** 클릭

**완료!** 이제 바로 OAuth 인증을 다시 시도할 수 있습니다.

---

#### 방법 B: 앱을 프로덕션으로 게시 (선택사항)

⚠️ **주의**: 이 방법은 Google 검토가 필요할 수 있어 시간이 걸립니다.

1. OAuth 동의 화면 상단에서
2. **"앱 게시"** 또는 **"PUBLISH APP"** 버튼 클릭
3. 경고 메시지 확인 후 **"확인"** 클릭

**권장하지 않음**: 개발 단계에서는 테스트 모드로 유지하고 테스트 사용자만 추가하는 것이 좋습니다.

---

## ❓ 자주 묻는 질문

### Q: 테스트 사용자는 몇 명까지 추가할 수 있나요?

**A**: 테스트 모드에서는 최대 **100명**까지 추가할 수 있습니다.

### Q: 여러 이메일을 한 번에 추가할 수 있나요?

**A**: 네, 쉼표나 엔터로 구분하여 여러 이메일을 한 번에 입력할 수 있습니다.

### Q: Gmail이 아닌 다른 Google 계정도 가능한가요?

**A**: 네, Google Workspace 계정(예: @company.com)도 가능합니다.

### Q: 테스트 사용자를 추가하면 바로 적용되나요?

**A**: 네, 즉시 적용됩니다. 몇 초 후 바로 로그인할 수 있습니다.

### Q: 프로덕션으로 게시해야 하나요?

**A**: 아니요. 개발/내부 사용 목적이라면 테스트 모드로 충분합니다.
프로덕션 게시는 Google 검토가 필요하며, 외부 사용자에게 공개할 때만 필요합니다.
