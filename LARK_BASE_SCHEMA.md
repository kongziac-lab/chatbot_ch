# Lark Base FAQ 테이블 스키마 가이드

이 프로젝트는 Lark Base FAQ 테이블에서 아래 필드명을 **정확히 동일하게** 사용합니다.

## 필수 필드 (FAQ 테이블)
- `고유번호` (텍스트)
- `카테고리(대분류)` (텍스트 또는 단일 선택)
- `카테고리(중분류)` (텍스트 또는 단일 선택)
- `질문(한국어)` (긴 텍스트)
- `답변(한국어)` (긴 텍스트)
- `질문(중국어)` (긴 텍스트)
- `답변(중국어)` (긴 텍스트)
- `출처` (텍스트)
- `상태` (단일 선택, 예: `게시중`)
- `생성부서` (텍스트)
- `적용범위` (텍스트)
- `생성일` (텍스트 또는 날짜)
- `수정일` (텍스트 ISO8601 권장)
- `우선순위` (숫자)
- `조회수` (숫자)
- `도움됨비율` (숫자)

## 선택 테이블
- 피드백 테이블(`LARK_FEEDBACK_TABLE_ID`)
- 문서 메타 테이블(`LARK_SOURCE_DOC_TABLE_ID`)

## 점검 방법
1. 필드 호환성 API:
- `GET /admin/lark-field-compat`

2. 스크립트 점검:
```bash
python3 scripts/lark_preflight.py
```

필드명이 하나라도 다르면 FAQ 조회/동기화가 실패할 수 있습니다.
